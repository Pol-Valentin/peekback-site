Features

Everything you need to close the feedback loop

🔍

DOM Inspector

Hover to highlight, click to select. Extracts a multi-signal fingerprint — CSS selector, outerHTML, textContent, attributes, React/Vue component name — so Claude knows exactly which element you mean.

📸

Area Screenshots

Draw a selection rectangle on the page. The cropped screenshot is saved as a PNG file that Claude can visually inspect with its Read tool. No more describing layouts in words.

💬

Bidirectional Chat

Send feedback, get replies — in the sidebar. Unlike one-way annotation tools, Peekback is a real conversation channel. Claude responds inline with context.

🔦

Remote Highlights

Claude can highlight elements on your page after making changes. Dismiss individually or toggle all off. See what changed without digging through diffs.

🔄

Multi-Session Hub

Run multiple Claude Code sessions? The first channel becomes the hub; others connect as clients. If the hub dies, a client takes over. One port, zero configuration.

🦊

Chrome + Firefox

Single codebase, dual manifests. Works on Chrome (Side Panel API) and Firefox (Sidebar Action) with CSP override for WebSocket localhost connections.

Quick Start

Up and running in 3 minutes

1

Clone and install

git clone https://github.com/Pol-Valentin/claude-feedback-browser-ext
cd claude-feedback-browser-ext
npm install
2

Build the extension

./scripts/build.sh
# Generates dist/chrome/ and dist/firefox/
3

Load in your browser

Firefox: about:debugging → Load Temporary Add-on → dist/firefox/manifest.json

Chrome: chrome://extensions → Developer Mode → Load unpacked → dist/chrome/

4

Launch Claude Code

claude --dangerously-load-development-channels server:claude-feedback
