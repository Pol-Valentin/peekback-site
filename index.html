// How it works

Three gestures,
zero context-switching

1
🔍

Inspect

Hover to highlight, click to select. The extension extracts a fingerprint — selector, HTML, text, attributes, React/Vue component name — so the AI knows exactly which element you mean.

2
💬

Comment

Type your feedback in the sidebar. "Make this red." "This button overflows on mobile." Or just capture an area screenshot and let the AI see what you see.

3
🔦

Get feedback

Claude replies in your sidebar and can highlight the elements it changed directly on the page. Dismiss highlights individually or toggle them all off.

// Features

Built for the feedback loop

Architecture

Bidirectional MCP Channel

Not a one-way annotation tool. Peekback uses Claude's experimental channel protocol for push notifications. Your feedback arrives instantly — no polling, no tool calls, no copy-paste.

Capture

Area Screenshots

Draw a rectangle, capture just that area. The PNG is saved to disk so Claude can visually inspect it. No more describing broken layouts in prose.

Scaling

Multi-Session Hub

Multiple Claude Code sessions? The first channel becomes the hub. Others connect as clients. If the hub dies, a client takes over. One port, zero config.

Browsers

Chrome + Firefox

Single codebase, dual manifests. Side Panel on Chrome, Sidebar Action on Firefox. CSP override for WebSocket localhost in Firefox MV3.

Detection

Framework-Aware

Detects React components via fiber internals and Vue instances via __vue__. In dev mode, the AI gets the component name — mapping straight to your source files.

// Quick start

Running in 3 minutes

1

Clone & install

git clone https://github.com/Pol-Valentin/claude-feedback-browser-ext cd claude-feedback-browser-ext && npm install
2

Build

./scripts/build.sh # → dist/chrome/ + dist/firefox/
3

Load in browser

Firefox: about:debugging → Load Temporary Add-on
Chrome: chrome://extensions → Developer Mode → Load unpacked

4

Launch Claude Code

claude --dangerously-load-development-channels server:claude-feedback
