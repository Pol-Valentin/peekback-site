Privacy Policy

Last updated: March 28, 2026

Overview

Peekback is an open-source browser extension that sends visual feedback (DOM element data and screenshots) from your browser to a locally running MCP channel connected to Claude Code. Your data never leaves your machine.

Data Collection

Peekback does not collect, store, or transmit any personal data to external servers. Specifically:

  • No analytics or telemetry
  • No cookies or tracking
  • No user accounts or authentication
  • No data sent to third-party services

Data Processed Locally

When you use Peekback, the following data is processed entirely on your local machine:

  • DOM element fingerprints — CSS selectors, HTML snippets, text content, and attributes of elements you select
  • Screenshots — PNG images captured from your browser viewport, saved temporarily in /tmp/
  • Messages — Text comments you write in the sidebar

This data is transmitted via WebSocket to a locally running MCP channel process (ws://localhost:9420) and then passed to Claude Code via MCP stdio. All communication happens on localhost — nothing is sent over the internet by the extension.

Permissions

The extension requests the following browser permissions:

  • activeTab / tabs — To capture screenshots and inject the content script
  • storage — To persist user preferences (highlight toggle state)
  • sidePanel (Chrome) — To display the sidebar UI

Third-Party Services

Peekback does not integrate with any third-party services. The MCP channel communicates exclusively with a locally running Claude Code instance.

Open Source

The complete source code is available at github.com/Pol-Valentin/claude-feedback-browser-ext. You can audit every line of code.

Contact

For questions about this privacy policy, open an issue on the GitHub repository.
